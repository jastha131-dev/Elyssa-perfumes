# Dynamic Page Builder — Design Spec

**Date:** 2026-05-15
**Project:** Luxe Parfum (Next.js 15 + Sanity CMS)
**Status:** Approved

---

## Goal

Give admin a Shopify-style homepage builder: add, remove, reorder, and configure sections visually with live preview — all from Sanity Studio.

---

## Architecture

```
Admin (/studio)
  └── Sanity Presentation Tool
        ├── LEFT: Form (block list, drag-reorder, per-block fields)
        └── RIGHT: Live preview iframe (homepage with draft data)

homePage Sanity document
  └── sections[] ← polymorphic array of typed blocks
        each block: _type, isVisible, content fields

Next.js homepage
  └── PageBuilder component
        maps block._type → React component
        filters isVisible: false
        renders in array order

Draft mode
  └── /api/draft-mode/enable — enables Next.js draft mode
      Presentation tool iframe uses draft mode automatically
```

## Sanity Schema

### Shared Object: `ctaButton`

| Field | Type | Notes |
|---|---|---|
| label | string | Button text |
| link | string | URL or path |
| style | string (enum) | `primary` \| `secondary` \| `ghost` \| `outline` |

### `homePage` document

Replaces current flat fields with a single `sections` array of polymorphic blocks.

### Block Types

| `_type` | Key Fields |
|---|---|
| `heroSection` | headline, subheadline, bgImage, bgVideo (optional), cta (ctaButton), textColor |
| `customBannerSection` | image, headline, subtext, overlayOpacity (0–100), cta (ctaButton), textAlign (left/center/right) |
| `featuredProductsSection` | title, products[] (product refs), layout (grid/carousel) |
| `bestSellersSection` | title, products[] (product refs, optional — auto-fetches if empty) |
| `categoriesSection` | title, categories[] (category refs) |
| `marqueeSection` | text, speed (number) |
| `scentBannerSection` | headline, subtext, bgImage, cta (ctaButton) |
| `brandStorySection` | headline, body (Portable Text), image, imagePosition (left/right) |
| `testimonialsSection` | title, testimonials[] (testimonial refs) |
| `newsletterSection` | headline, subtext, buttonLabel, bgImage (optional) |
| `trustBarSection` | items[] (icon, label, value) |

Every block includes:
- `isVisible` (boolean, default true) — toggle without deleting
- `_key` (auto-generated by Sanity) — used as React key

All image fields: admin can upload new asset or pick from existing Sanity media library.

## Frontend

### `PageBuilder` component

New component at `components/PageBuilder.tsx`. Replaces 10 hardcoded section imports in `app/page.tsx`.

```tsx
const blockMap = {
  heroSection: Hero,
  customBannerSection: CustomBanner,
  featuredProductsSection: FeaturedProducts,
  bestSellersSection: BestSellers,
  categoriesSection: Categories,
  marqueeSection: MarqueeStrip,
  scentBannerSection: ScentBanner,
  brandStorySection: BrandStory,
  testimonialsSection: Testimonials,
  newsletterSection: Newsletter,
  trustBarSection: TrustBar,
}

export function PageBuilder({ sections }) {
  return sections
    .filter(s => s.isVisible !== false)
    .map(s => {
      const Block = blockMap[s._type]
      return Block ? <Block key={s._key} data={s} /> : null
    })
}
```

### Existing component updates

Each existing component (Hero, FeaturedProducts, BestSellers, etc.) updated to accept a `data` prop from Sanity instead of fetching data independently. Visual output unchanged.

### New component

`components/home/CustomBanner.tsx` — new block, built fresh. Renders full-width image with text overlay, configurable opacity and alignment, and CTA button.

### `app/page.tsx`

Simplified to fetch `homePage.sections` from Sanity, pass to `<PageBuilder sections={data.sections} />`. Uses `draftModeClient` when `draftMode().isEnabled`.

## Sanity Studio Config

### Install

```bash
npm install @sanity/presentation
```

### `sanity.config.ts`

```ts
import { presentationTool } from '@sanity/presentation'

// Add to plugins array:
presentationTool({
  previewUrl: {
    origin: process.env.NEXT_PUBLIC_SITE_URL,
    draftMode: { enable: '/api/draft-mode/enable' }
  }
})
```

## Draft Mode API Routes

### `app/api/draft-mode/enable/route.ts`

- Validates `?secret=` query param against `SANITY_PREVIEW_SECRET`
- Calls `draftMode().enable()`
- Redirects to homepage (or `?redirectTo` param)

### `app/api/draft-mode/disable/route.ts`

- Calls `draftMode().disable()`
- Redirects to homepage

## Environment Variables

Add to `.env.local`:

```
SANITY_PREVIEW_SECRET=<random string, min 32 chars>
NEXT_PUBLIC_SITE_URL=http://localhost:3000  # or production URL
```

## Data Flow (Admin Workflow)

1. Admin opens `/studio` → clicks **Presentation** tab
2. Right iframe shows live homepage with draft data
3. Admin clicks **+** to add a block, picks type from menu
4. Admin fills fields in left panel — preview updates in real time
5. Admin drags blocks to reorder — homepage reorders in preview
6. Admin toggles `isVisible` off — section disappears in preview
7. Admin clicks **Publish** — live site updates
